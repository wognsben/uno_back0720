# Legacy Admin Product Editor Gap Audit

## 결론

현재 리뉴얼 상품 에디터는 기존 어드민 기능을 너무 한 화면에 몰아넣었습니다. 특히 `운영 옵션`, `세미패키지 일정`, `데일리투어 캘린더`, `요금/특가`, `상세 이미지/코스`는 기존에도 별도 화면 또는 팝업으로 분리되어 있었고, 리뉴얼에서도 분리하는 편이 맞습니다.

상품 수정 본문은 “핵심 상품 정보와 이동 허브”가 되어야 하고, 복잡한 일정/캘린더/요금/상세 콘텐츠 관리는 팝업 또는 별도 패널로 열어야 합니다.

## 현재 리뉴얼 에디터 누락/오류

| 영역 | 현재 문제 | 맞는 방향 |
| --- | --- | --- |
| 공통 관리자 레이아웃 | 일부 리뉴얼 페이지가 헤더/좌측 메뉴 없는 단독 페이지처럼 보임 | 모든 관리자 화면에 공통 헤더, 상단 메뉴, 좌측 메뉴 유지 |
| 운영 옵션 | 상품 수정 화면 안에 HTML 텍스트처럼 노출됨 | `투어 옵션` 팝업/모달로 분리. HTML을 그대로 보여주기보다 줄바꿈 기반 입력과 미리보기 제공 |
| 세미 보딩패스 | 항공권 관리처럼 좌석/추가금/상태가 과하게 보임 | 프런트의 보딩패스 카드에 필요한 출발지/도착지/출발일시/도착일시 중심으로 단순화 |
| 세미 일차별 일정 | 상품 수정 본문에 섞이면 가독성 낮음 | `일정 1일차`, `특전`, `이용방법`, `포함`, `불포함`, `주의사항`을 팝업으로 분리 |
| 데일리 캘린더 | 행 리스트처럼 보이거나 0 입력 두 개가 의미 불명확 | 한 달 캘린더, 이전/다음 월 버튼, 상태/정원/예약/잔여 라벨 명확화 |
| 데일리 요금 옵션 | 입력칸 잘림, 정상가/특가/할인가 기능 부족 | 기본 요금과 날짜별 특가/할인가/정상가를 별도 요금 모달로 분리 |
| 가이드 지정 | 현재 리뉴얼 상세 편집에서 빠짐 | `guide_info` 다중 선택 지원, 프런트 상세 하단 가이드 노출과 연결 |
| 추천투어/다음 투어 | 일부만 연결됨 | `recommend_tour` 및 `wr_event_course`를 구분해 관리 |
| 정상 가격 문구 | `fee_org` 누락 | 상품 목록/상세 요금 하단 정상가 문구 관리 필요 |
| 예약 문구 | 상단/중단/하단/이벤트/바우처/환불/상세 알림이 모두 분리되어 있음 | 예약/바우처/상세 알림 섹션으로 묶어 관리 |
| 여권/룸/배송 필요 여부 | 일부 상품 타입만 반영됨 | `is_passport`, `is_roominfo`, `is_delivery`를 명시적으로 관리 |
| 상세 콘텐츠 | PC/Mobile 상단 이미지, 코스, Info, 광고, 배너 관리 누락 | `상세 콘텐츠 관리` 별도 화면/모달로 분리 |

## 상품 수정 페이지 새 구조

### 1. 상품 기본

- 상품명
- 카테고리
- 프런트 노출 상태: `노출`, `준비중`, `숨김`
- 정렬
- 대표 productId / legacy id
- 티켓 여부
- B2B 예약 기본 상태
- 일반 예약 기본 상태
- 여권/룸/배송 필요 여부

### 2. 요금 요약

- 대표 예약금
- 대표 현지지불금
- 정상 가격 문구 `fee_org`
- 요금 옵션 관리 버튼
- 데일리투어일 경우 날짜별 특가/할인가/정상가 관리 버튼

### 3. 예약/문구

- 예약 페이지 상단 문구 `reg_msg_top`
- 예약 페이지 중단 문구 `reg_msg_middel`
- 예약 페이지 하단 문구 `reg_msg_bottom`
- 이벤트 문구 `reg_msg_event`
- 바우처 문구 `voucher_msg`
- 상세 환불규정 `wr_can_rule`
- 상세 알림 팝업 문구 `pop_content`

### 4. 프런트 연결

- 가이드 지정 `guide_info`
- 추천투어 `recommend_tour`
- 이벤트/연관 코스 `wr_event_course`
- 선택된 여행지 `wr_9`
- 상품 네비게이션 노출 그룹

### 5. 관리 버튼 허브

- 투어 옵션 관리
- 상세 이미지/코스/Info 관리
- 세미패키지 일정 카드 관리
- 세미패키지 일차별 일정 관리
- 데일리투어 캘린더 관리
- 데일리투어 요금/특가 관리

## 팝업/모달 분리 원칙

| 팝업/모달 | 대상 | 이유 |
| --- | --- | --- |
| 투어 옵션 관리 | 모임장소, 모임시간, 투어시간, 포함/불포함, 준비물, 취소규정 | 긴 문장과 이미지가 섞여 상품 기본 정보와 분리 필요 |
| 상세 콘텐츠 관리 | 상단 이미지, 코스 이미지, Info 이미지, 모바일 이미지, 배너 링크 | 파일 업로드와 미리보기가 많아 별도 화면이 적합 |
| 세미 일정 카드 관리 | 출발지/도착지/출발일시/도착일시, 예약금/중도금/잔금, 노출 | 프런트 보딩패스 표시 데이터. 항공권이 아니라 일정 카드로 명명 |
| 세미 일차별 일정 | 일차별 도시/일정/식사/호텔, 특전, 이용방법, 포함/불포함, 주의사항 | 기존 `pkgSkd` 팝업 흐름 유지 |
| 데일리 캘린더 관리 | 월별 투어 가능/마감/정원/예약/잔여 | 한 달 캘린더 UI가 가장 이해 쉬움 |
| 데일리 요금 관리 | 기본 요금, 날짜별 특가/할인가/정상가 | 요금 단위와 기간 적용이 복잡해 별도 관리 필요 |

## 우선순위

1. 리뉴얼 관리자 공통 레이아웃부터 만든다.
2. 현재 상품 에디터에서 잘못 들어간 인라인 운영 옵션/세미 일정/데일리 캘린더를 분리한다.
3. 상품 수정 본문은 누락 필드가 없는 “요약 + 관리 버튼 허브”로 재구성한다.
4. 데일리 캘린더 팝업을 한 달 단위 UI로 완성한다.
5. 세미 일정 카드 팝업을 프런트 보딩패스와 동일한 언어로 재설계한다.
6. 상세 콘텐츠/운영 옵션/요금 관리 팝업을 기존 저장 테이블과 1:1로 연결한다.

## 다음 구현 기준

- 기존 데이터 테이블은 최대한 유지한다.
- 사용자에게 보이는 용어는 항공권이 아니라 `일정 카드`, `출발 일정`, `도착 일정`으로 정리한다.
- 관리자는 한 화면에서 모든 걸 입력하는 것이 아니라, 상품 수정 페이지에서 필요한 관리 화면을 빠르게 여는 방식으로 만든다.
- 새 PHP 파일은 기존처럼 `admin/include/header.php` 또는 리뉴얼 공통 헤더를 반드시 포함한다.
